### 一、 核心要素：状态与动作 (State & Action)

- **状态 (State, $s$)**：智能体相对于环境的状态 。在网格世界案例中，状态即为智能体的坐标位置（如 $s_1 \dots s_9$）。
    
- **状态空间 (State Space, $\mathcal{S}$)**：所有可能状态的集合 。
    
- **动作 (Action, $a$)**：智能体在某一状态下可执行的操作 。
    
- **动作空间 (Action Space, $\mathcal{A}(s)$)**：特定状态 $s$ 下所有可用动作的集合 。**注意**：不同状态可以拥有不同的动作空间 。
    

### 二、 状态转移 (State Transition)

智能体执行动作后，从当前状态转移到下一状态的过程 。

- **描述方法**：
    
    1. **表格法**：纵轴为状态，横轴为动作。这种方法直观但只能描述**确定性**的转移 。
        
    2. **数学概率法**：使用条件概率 $p(s'|s, a)$ 描述 。这种方式更为通用，可以描述**随机性**转移（例如受风力干扰导致移动偏离预期） 。
        

### 三、 策略 (Policy)

策略定义了智能体在特定状态下选择动作的规则 。

- **描述方法**：
    
    1. **直观描述**：在环境地图中使用箭头标示动作方向 。
        
    2. **表格描述**：列出每个状态对应的动作，可涵盖确定性和随机性策略 。
        
    3. **数学概率法**：使用条件概率 $\pi(a|s)$ 表示在状态 $s$ 下采取动作 $a$ 的概率 。
        

### 四、 奖励 (Reward)

奖励是智能体执行动作后获得的即时数值反馈，是实现人机交互、引导智能体行为的关键手段 。

- **性质**：正数代表鼓励，负数代表惩罚，零代表无惩罚 。
    
- **描述方法**：
    
    1. **表格法**：展示状态-动作对对应的固定奖励值（仅限确定性奖励） 。
        
    2. **条件概率 $p(r|s, a)$**：描述获得特定奖励值的概率（可处理随机奖励，如“努力学习不一定能拿高分”） 。
        

### 五、 轨迹、回报与回合 (Trajectory, Return & Episode)

- **轨迹 (Trajectory)**：一个“状态-动作-奖励”的交互链条 。
    
- **回报 (Return)**：一条轨迹上所有即时奖励的总和，是评价策略优劣的量化指标 。
    
- **折扣回报 (Discounted Return)**：为解决轨迹无限长导致回报发散的问题，引入折扣因子 $\gamma \in [0, 1)$ 。
    
    - 公式：$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \dots$
        
- **回合 (Episode)**：从初始状态开始到终止状态结束的一个完整交互过程（即一个 Trial） 。
    

### 六、 马尔可夫决策过程 (MDP)

- **马尔可夫性 (Markov Property)**：一种无记忆性质，即下一时刻的状态和奖励仅取决于当前时刻的状态和动作，而与历史过程无关 。
    
- **MDP 的构成**：包括集合（$\mathcal{S, A, R}$）和概率分布（转移概率、奖励概率、策略） 。
    
- **与马尔可夫过程 (MP) 的关系**：一旦策略 $\pi$ 被固定，智能体的动作选择就变得确定或符合固定分布，此时 MDP 便退化为**马尔可夫过程 (Markov Process)** 。
