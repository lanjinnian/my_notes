## 一、 价值迭代算法 (Value Iteration Algorithm)

价值迭代算法旨在通过迭代方式求解贝尔曼最优方程，从而找到最优状态价值和最优策略 。

- **理论依据**：该算法是基于压缩映射定理（Contraction Mapping Theorem）提出的一种迭代算法 。
    
- **核心公式**：在给定任意初始值 $v_{0}$ 的情况下，迭代公式为 $v_{k+1} = \max_{\pi}(r_{\pi} + \gamma P_{\pi}v_{k})$ 。
    
- **算法拆解**：迭代过程可以拆分为以下两个主要步骤 。
    
- **步骤 1 - 策略更新 (Policy Update)**：在已知 $v_{k}$ 的情况下，求解 $\pi_{k+1} = \arg\max_{\pi}(r_{\pi} + \gamma P_{\pi}v_{k})$ 。
    
- **步骤 1 (元素级表示)**：通过计算动作价值（q-value） $q_{k}(s,a) = \sum_{r}p(r|s,a)r + \gamma\sum_{s^{\prime}}p(s^{\prime}|s,a)v_{k}(s^{\prime})$ 来找到最大动作价值的动作 。生成的策略是一个贪婪策略（greedy policy），即当 $a = a_{k}^{*}(s)$ 时概率为 1，否则为 0 。
    
- **步骤 2 - 价值更新 (Value Update)**：更新状态价值公式为 $v_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}}v_{k}$ 。
    
- **步骤 2 (元素级表示)**：由于 $\pi_{k+1}$ 是贪婪策略，该公式可以直接简化为 $v_{k+1}(s) = \max_{a}q_{k}(a,s)$ 。需要注意的是，此处的 $v_{k}$ 并非严格意义上的“状态价值”，因为它不一定满足贝尔曼方程 , 可能是任意的一个向量。
    
- **停止条件**：当两次迭代的价值差 $||v_{k} - v_{k-1}||$ 小于预设的微小阈值时, 认为$v _k$已经收敛了，算法停止并输出结果 。

## 二、 策略迭代算法 (Policy Iteration Algorithm)

策略迭代算法通过反复交替进行策略评估与策略改进，从一个随机的初始策略 $\pi_{0}$ 逐步逼近最优策略 。

- **步骤 1 - 策略评估 (Policy Evaluation, PE)**：计算当前策略 $\pi_{k}$ 的状态价值函数，公式为 $v_{\pi_{k}} = r_{\pi_{k}} + \gamma P_{\pi_{k}}v_{\pi_{k}}$ 。
    
- **策略评估的求解方法**：可以通过闭式解 $v_{\pi_{k}} = (I - \gamma P_{\pi_{k}})^{-1}r_{\pi_{k}}$ 直接求解，或者使用嵌套的迭代公式 $v_{\pi_{k}}^{(j+1)} = r_{\pi_{k}} + \gamma P_{\pi_{k}}v_{\pi_{k}}^{(j)}$ 进行求解 。这说明策略迭代的评估步骤内部本身就包含着另一个迭代算法 。
    
- **步骤 2 - 策略改进 (Policy Improvement, PI)**：基于评估得到的价值更新策略，公式为 $\pi_{k+1} = \arg\max_{\pi}(r_{\pi} + \gamma P_{\pi}v_{\pi_{k}})$ 。
    
- **策略改进定理**：该定理保证了每次改进后的策略价值不会低于前一次，即 $v_{\pi_{k+1}} \ge v_{\pi_{k}}$ 。  

> 直观理解: 策略改进的选择方式实际上是一种稳赚不赔, 最差也是打平的贪心选择过程
    
- **算法收敛性**：随着每次迭代都会提升策略，状态价值序列会持续增加，并最终收敛于最优状态价值 $v^{*}$，同时策略序列也会收敛于最优策略 。

- 接近目标的策略会先变好, 原理目标的策略后变好

## 三、 价值迭代与策略迭代的比较

这两种算法在结构上高度相似，都具备相同的初始条件，且前三步（策略初始化、价值计算、策略更新）本质上是相同的 。

- **计算深度的差异**：两者的核心区别在于价值更新的计算深度 。
    
- **价值迭代的特点**：价值更新是一个单步计算，直接通过 $v_{1} = r_{\pi_{1}} + \gamma P_{\pi_{1}}v_{0}$ 完成 。
    
- **策略迭代的特点**：为了求解精确的状态价值 $v_{\pi_{1}}$，策略评估步骤在理论上需要进行无限次的内部迭代 。

> [!NOTE]
> - 策略迭代: 算出一个**规则**的真实成绩，再去换新**规则**
> - 价值迭代: 假定每次都做**最优动作**去刷新最高**成绩**，最后才提炼出规则

## 四、 截断策略迭代算法 (Truncated Policy Iteration Algorithm)

为了解决策略迭代中策略评估步骤需要无限次迭代的问题，截断策略迭代引入了有限迭代次数（即截断）的机制 。

- **核心机制**：在策略评估中，不追求完全收敛，而是将最大迭代次数限制为一个预设的常数（截断值） 。当内部迭代次数达到该限制后，直接将计算结果赋值给当前状态价值 $v_{k}$ 。
    
- **初始猜测的优化**：将上一次迭代得到的状态价值作为当前策略评估的初始猜测，即 $v_{\pi_{k}}^{(0)} = v_{\pi_{k-1}}$ 。
    
- **价值提升与收敛性 (Value Improvement)**：截断操作并不会破坏算法的收敛性 。命题证明，如果初始猜测为 $v_{\pi_{k-1}}$，那么对于每次内部迭代，价值依然能保持递增状态（即 $v_{\pi_{k}}^{(j+1)} \ge v_{\pi_{k}}^{(j)}$） 。此外，由于价值迭代能确保收敛，策略迭代及其截断版本自然也能保证收敛 。
    
- **截断参数与性能表现**：增加策略评估步骤的截断次数（例如从 1 次增加到 3 次、6 次甚至 100 次）可以加快价值估计的收敛速度 。然而，当截断次数较大时，增加次数所带来的边际效益会迅速衰减 。因此在实际应用中，策略评估步骤只需运行少量的几次迭代即可达到理想效果 。

> [!NOTE]
> 在数学本质上，价值迭代其实就是被“极端截断”的策略迭代。它们不是两个完全不同的算法，而是同一条线段上的两个端点。
