## 向上取整:math.ceil(x)
返回 _x_ 的向上取整，即大于或等于 _x_ 的最小的整数。
> x必须是浮点数

## 绝对值:math.fabs(_x_)
返回 _x_ 的绝对值。

## 向下取整math.floor(_x_)
返回 _x_ 的向下取整，小于或等于 _x_ 的最大整数。
> x必须是浮点数

## 融合乘加:math.fma(_x_, _y_, _z_)
融合的乘法-加法运算。 返回 `(x * y) + z`，类似于使用无限精度和取值范围进行计算然后执行一次舍入到 `float` 格式。 此运算往往可提供比直接使用表达式 `(x * y) + z` **更高的精确度**。
> 3.13后的版本才能使用
> 也许可以用来算偏置和权重(?)

## 浮点余数:math.fmod(_x_, _y_)

## 分离小数与整数:math.modf(_x_)
返回 _x_ 的小数和整数部分。两个结果都带有 _x_ 的符号并且是浮点数。
> 返回的元组顺序是 `(小数部分, 整数部分)`
> 返回的两个值都是 **浮点数**, 正负号均继承

math.remainder(_x_, _y_)
返回 IEEE 754 风格的 _x_ 相对于 _y_ 的余数。对于有限 _x_ 和有限非零 _y_ ，这是差异 `x - n*y` ，其中 `n` 是与商 `x / y` 的精确值最接近的整数。如果 `x / y` 恰好位于两个连续整数之间，则将最接近的 _偶数_ 用作 `n` 。 余数 `r = remainder(x, y)` 因此总是满足 `abs(r) <= 0.5 * abs(y)`。

特殊情况遵循 IEEE 754：特别是 `remainder(x, math.inf)` 对于任何有限 _x_ 都是 _x_，而 `remainder(x, 0)` 和 `remainder(math.inf, x)` 对于任何非 NaN 的 _x_ 会引发 [`ValueError`](https://docs.python.org/zh-cn/3/library/exceptions.html#ValueError "ValueError")。如果余数运算的结果为零，则该零将具有与 _x_ 相同的符号。

在使用IEEE 754二进制浮点的平台上，此操作的结果始终可以完全表示：不会引入舍入错误。

Added in version 3.7.

math.trunc(_x_)[](https://docs.python.org/zh-cn/3/library/math.html#math.trunc "Link to this definition")

返回去除小数部分的 _x_ ，只留下整数部分。 这会向 0 舍入： `trunc()` 对于正的 _x_ 相当于 [`floor()`](https://docs.python.org/zh-cn/3/library/math.html#math.floor "math.floor") ，对于负的 _x_ 相当于 [`ceil()`](https://docs.python.org/zh-cn/3/library/math.html#math.ceil "math.ceil") 。如果 _x_ 不是浮点数，委托给 [`x.__trunc__`](https://docs.python.org/zh-cn/3/reference/datamodel.html#object.__trunc__ "object.__trunc__") ，它应该返回一个 [`Integral`](https://docs.python.org/zh-cn/3/library/numbers.html#numbers.Integral "numbers.Integral") 值。

对于 [`ceil()`](https://docs.python.org/zh-cn/3/library/math.html#math.ceil "math.ceil") ， [`floor()`](https://docs.python.org/zh-cn/3/library/math.html#math.floor "math.floor") 和 [`modf()`](https://docs.python.org/zh-cn/3/library/math.html#math.modf "math.modf") 函数，请注意 _所有_ 足够大的浮点数都是精确整数。Python 浮点数通常不超过53位的精度（与平台 C double 类型相同），在这种情况下，任何满足 `abs(x) >= 2**52` 的浮点数 _x_ 必然没有小数位。
